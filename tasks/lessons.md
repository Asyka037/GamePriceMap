# Lessons

## 2026-07-08
- 即使计划写着"今天实测"，执行前仍按当前日期重新核验关键外部端点、免费额度和返回字段；外部商店 API、折扣数量和平台限额都是易变事实。
- 不要把参考项目的 `validate-data.mjs` 等同于生产校验闸门；参考实现只证明数据形状可读，正式管线还需要价格范围、折扣范围、区域覆盖率、汇率换算和失败保旧数据等断言。
- CheapShark 会拒绝通用 User-Agent；后续所有抓取脚本必须统一使用项目级 HTTP helper 注入真实描述性 UA，避免某个脚本绕过公共 helper。
- 文档中的实时数量（如 Nintendo Europe 折扣条目数）只能作为核验样例，不应作为稳定验收标准。
- 记录外部端点行为时必须带上**触发条件和测试日期**，否则复核会产生假冲突：Steam appdetails 组合 filters 单 appid 是 200、批量 appids 是 400——"组合 filters 会 400"和"组合 filters 能用"都对了一半。复核他人的端点结论时，先还原对方的请求条件（单/批量、参数、UA）再下结论。
- CheapShark `cheapestPriceEver` 会把 Epic 限免等赠送记录当成 $0.00 史低返回；聚合器语义里"赠送"不等于"售价"，任何外部史低种子必须拒绝 ≤0 的值。发现方式：validate 闸门的 `atl>0` 断言在首日就拦住了两条脏数据——校验器先于直觉。
- NSUID 发现的两类真实错配：franchise 名是续作标题的前缀（"Hades" startsWith 匹配到 "HADES II"），标题匹配必须用精确相等而非 contains/startsWith；猜测的商品页 URL 会 200 重定向到通用商店页并携带无关游戏的 nsuid，必须校验最终 URL + 页面含目标标题 + 跨游戏 nsuid 去重。半自动发现管线里"宁可落空转人工，不可低置信度自动写入"。
- Nintendo price API 会返回多年未调的通胀遗留价（AR 区 2017 年 ARS 定价换算后 $0.12），属于"真实返回但经济上无意义"的数据；聚合层需要按中位数比例过滤离群区域，且过滤逻辑要做成带单测的纯函数。

## 2026-07-09
- Twitch 官方 OAuth 文档仍用 `http://localhost:3000` 举例，但当前开发者控制台注册表单会提示“重定向 URL 必须使用 HTTPS”；操作指引应以实时控制台校验为准，并说明文档滞后。DealDex 访问 IGDB 使用 Client Credentials，不发送 `redirect_uri`，因此可在注册表单填 `https://localhost` 作为未使用的占位地址；只有未来真正做用户 OAuth 登录时，才需要换成可访问且精确匹配的 HTTPS 回调。
- 构建期读文件不要用 fs + import.meta.url 相对路径：Astro/Vite 打包后模块位置改变，运行时才 ENOENT。构建期静态数据的通用形态是 ESM 数据模块（export default {...}），Node 测试与打包器都天然支持。
- 地理质心要取最大连通块而非全体格子：阿拉斯加把 US 全格质心拉到加拿大边界，视觉锚点类特征永远考虑离岛/飞地。

## 2026-07-10
- 用户明确要求：非代码型研究、方案评审和会话结论不写入 `tasks/todo.md`；该文件只追踪实际实施工作。此类内容应在对话中交付，或仅在用户明确要求时写入专门文档。发现误写后立即撤回，不将其保留为任务记录。

## 2026-07-11
- 正则归一化 [^a-z0-9] 会把非拉丁文标题清成空串，两个空串"精确相等"——任何标题匹配必须用 Unicode 属性类（\p{Letter}\p{Number} + u flag）且空归一化不算命中。
- "只遍历现存文件"的校验器天然放行删除；必需产物集合要从 catalog（事实源）正向推导，而不是从磁盘反向枚举。
- 快照式数据源必然有"促销已结束但快照未刷新"的滞后窗口；展示层必须用 endsAt 做集中过期过滤，不能只依赖抓取频率。
- 跨渠道聚合值（overallAtl）放进渠道行是语义错误的模板陷阱：任何"每行一个渠道"的表格只能用该渠道自己的派生值。
- 框架便利 API（Astro define:vars）可能自带漏洞面（GHSA-j687-52p2-xcff）；向内联脚本传外部数据的通用安全形态是"转义 < 的 JSON 数据脚本 + JSON.parse"。
- 存储层禁止入库派生值：把 USD 换算/排名写进快照，汇率第四位小数的日常抖动就会让"价格没变"的文件天天产生 diff（实证 42/42 全脏）。git 事实源只放原始观测（本币 + 元信息），派生留给构建期。
- 请求/带宽预算表必须显式标注维度（市场数×游戏数×页大小），单市场估算不标注会被按全区域读出数量级错误。
- 展示百分比时公式与措辞必须指向同一分母："X% below today" 的分母是今天价（1−atl/best，恒<100%），(best/atl−1) 算的是"今天高于史低多少"，两者在价差大时读数完全不同（实证 Elden Ring 显示 100% 而正确是 50%）。凡是模板句里的数字，单测要断言整句文案而不只断言分档；截图级预览验证能抓住纯逻辑测试漏掉的语义错配。

## 2026-07-12
- Microsoft Display Catalog 同一标准版 SKU 会同时返回正价 `Purchase` 与大量 `$0 License/Redeem` availability；绝不能取最低价格。主机商店解析必须按 edition/SKU/动作/有效期逐层筛选，只接受当前有效、含 `Purchase` 的正价 offer，订阅授权和试玩不等于免费购买。
- 新鲜度台账要区分“有旧数据可用”和“本轮完整成功”：失败区域行进位只能保证不写坏数据，不能据此推进 `lastSuccessAt`。失败若让脚本直接退出、从未提交台账，`consecutiveFailures` 也会形同虚设；独立渠道应保旧、记录失败并让后续 validate/commit 继续。
- 原始字段禁入规则应维护一个可测试的完整 denylist；只检查 `usd/rank` 会漏掉同属派生值的 `listUsd`。schema 约定写成复数字段时，测试也必须逐项锁定。
- 响应式网格禁止用行内 `grid-template-columns` 固定列宽：行内样式会压过媒体查询。桌面截图正常不代表移动端正常；对新增渠道造成的多列布局必须用真实窄视口同时断言无横向溢出和列已堆叠。
