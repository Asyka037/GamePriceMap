# AGENTS.md

## 工作流编排

### 自我改进循环
- 每次收到用户修改或纠正后：更新 `tasks/lessons.md` 并记录模式
- 为自己编写规则，防止再次犯同样错误
- 每次会话开始时，先复习与当前项目相关的教训

### 完成前必须验证
- 绝不在证明任务确实有效前标记“完成”
- 必要时对比主分支与你的改动
- 自问：“一个anthropic工程师会批准这个改动吗？”
- 运行测试、检查日志、演示正确性

### 追求优雅
- 对于非简单改动：暂停并询问“有没有更优雅的实现方式？”
- 如果修复感觉是 hack：用“我现在知道的所有信息，重新实现优雅方案”
- 简单、明显的修复跳过此步，不要过度工程化

### 自主修复 Bug
- 收到 Bug 报告时：直接修复，不要要求手把手指导
- 指向日志、错误、失败测试，然后解决它们
- 用户无需切换上下文
- 直接修复失败的 CI 测试，无需被告知如何操作

### 任务管理
1. **先规划**：把计划写到 `tasks/todo.md`，使用可勾选条目
2. **验证进度**：开始实现前先向我汇报
3. **跟踪进度**：实时标记已完成条目
4. **说明改动**：每一步给出高层摘要
5. **记录结果**：在 `tasks/todo.md` 中添加评审部分
6. **捕获教训**：修改后更新 `tasks/lessons.md`

## 多Agent 角色

- 本项目由Codex和CodeX共同维护
- 用户是唯一有权触发"推送到生产"的人。任何 Agent 均不得自行推送 `main`，除非用户在当前对话中明确要求。
